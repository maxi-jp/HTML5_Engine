HTML5_Engine Documentation

Engine and examples are actively hosted on GitHub Pages: https://maxi-jp.github.io/HTML5_Engine/

Core Concepts

The Game Class

The Game class (from engine/game.js) is the central hub of your game. It manages the game loop, holds references to all active game objects, and provides methods for initialization (Start), updating (Update), and drawing (Draw). Your main game logic will reside in a class that extends Game.

GameObjects

The GameObject class (from engine/gameobjects.js) is the base class for all interactive elements in your game. Every entity that needs to be updated or drawn (players, enemies, projectiles, UI elements) should inherit from GameObject or one of its specialized subclasses (e.g., RectangleGO, SpriteGO, AnimationGO).

class MyRotationBox extends RectangleGO {
    constructor(position) {
        // A 100x100 pixels blue box at the given position
        super(position, 100, 100, Color.blue);

        this.rotationSpeed = 1; // Radians per second
    }

    Update(deltaTime) {
        // Rotate the box over time
        this.rotation += this.rotationSpeed * deltaTime;
    }
    
    // The base RectangleGO class handles drawing the rectangle
}

Rendering

The Renderer class (from engine/renderer.js) provides methods for drawing shapes, sprites, and text onto the HTML5 Canvas. It's typically accessed via this.renderer within your Game class.

// Example of drawing in your Game's Draw method
    this.renderer.DrawFillRectangle(x, y, width, height, Color.red);
    this.renderer.DrawSprite(mySpriteImage, x, y, width, height);
    this.renderer.DrawText("Hello World", x, y, "20px Arial", Color.white);

The Game Loop

The game loop, managed by engine/main.js, continuously calls your game's Update and Draw methods. Update(deltaTime) is called with a deltaTime parameter, representing the time elapsed since the last frame, enabling frame-rate independent logic.
